# myservice

A production-ready Go service scaffold with structured logging, OpenTelemetry instrumentation, and PostgreSQL persistence.

## Stack

| Concern     | Choice                          |
|-------------|---------------------------------|
| Router      | chi v5                          |
| Database    | PostgreSQL via pgx/v5 + sqlc    |
| Migrations  | goose                           |
| Config      | envconfig (environment variables)|
| Logging     | log/slog (JSON in prod, text in dev) |
| Telemetry   | OpenTelemetry (traces + Prometheus metrics) |

## Prerequisites

- Go 1.23+
- PostgreSQL 15+
- [golangci-lint](https://golangci-lint.run/usage/install/)
- [sqlc](https://docs.sqlc.dev/en/latest/overview/install.html)
- [goose](https://github.com/pressly/goose#install)

## Quick start

```bash
# Copy and edit environment variables
cp .env.example .env

# Create the database
createdb myservice

# Run migrations
make migrate-up

# Build and run
make build
./myservice

# Verify
curl -s http://localhost:8080/healthz | jq .
```

## Make targets

| Target         | Description                            |
|----------------|----------------------------------------|
| `build`        | Compile binary to `./myservice`        |
| `run`          | Build and run the service              |
| `test`         | Run tests with race detector           |
| `lint`         | Run golangci-lint                      |
| `migrate-up`   | Apply all pending migrations           |
| `migrate-down` | Roll back the last migration           |
| `generate`     | Regenerate sqlc query code             |
| `docker-build` | Build Docker image                     |
| `docker-run`   | Build and run in Docker with `.env`    |
| `clean`        | Remove binary and test cache           |

## Environment variables

All configuration is read from environment variables. See [`.env.example`](.env.example) for the full list.

| Variable             | Default       | Description                     |
|----------------------|---------------|---------------------------------|
| `SERVICE_NAME`       | `myservice`   | Service identifier for logs/traces |
| `SERVICE_ENV`        | `dev`          | Environment (`dev`, `staging`, `prod`) |
| `DB_HOST`            | `localhost`   | PostgreSQL host                 |
| `DB_PORT`            | `5432`        | PostgreSQL port                 |
| `DB_USER`            | `postgres`    | PostgreSQL user                 |
| `DB_PASSWORD`        | `postgres`    | PostgreSQL password             |
| `DB_NAME`            | `myservice`   | PostgreSQL database name        |
| `DB_SSLMODE`         | `disable`     | PostgreSQL SSL mode             |
| `DB_MAX_CONNS`       | `10`          | Connection pool maximum         |
| `HTTP_PORT`          | `8080`        | HTTP listen port                |
| `HTTP_READ_TIMEOUT`  | `5s`          | HTTP read timeout               |
| `HTTP_WRITE_TIMEOUT` | `10s`         | HTTP write timeout              |
| `HTTP_IDLE_TIMEOUT`  | `120s`        | HTTP idle timeout               |
| `OTEL_ENABLED`       | `false`       | Enable OpenTelemetry            |
| `OTEL_SERVICE_NAME`  | `myservice`   | OTel service name               |
| `OTEL_METRICS_PORT`  | `9090`        | Prometheus metrics port         |
